## Publish Tasks

After the user approves the task breakdown:

1. Write all approved tracer-bullet tasks to `TASKS.md`.
2. Order tasks by dependency, with blockers first.
3. Give each task:
   - a number
   - a title
   - status
   - blocked-by field
   - end-to-end deliverable
   - acceptance criteria
   - verification commands
4. Use Markdown checkboxes for task status.
5. Do not create GitHub Issues.
6. Do not create one file per task.
7. Do not modify `PLAN.md`; treat it as the source specification.
8. Preserve completed tasks under a `Completed` section.
